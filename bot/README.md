# Discord Bot

A thin Discord interface to the FastAPI power-tracking backend in `../backend`. The bot owns **no device state** — it just fetches JSON from the backend, hands it to Groq for humanization, and posts the result.

## Commands

| Command | What it does |
|---|---|
| `!status` | Office-wide overview: which devices are ON/OFF, grouped by room, plus live total wattage. |
| `!room <name>` | Drill into one room (e.g. `!room work1`). |
| `!usage` | Live total wattage right now + today's estimated kWh and cost. |

Every reply is generated by Groq using a strict system prompt that **forbids hallucinated numbers, markdown tables, and code fences** — the LLM can only report what the backend JSON literally contains.

## Setup

```bash
cd bot
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Copy `.env.example` to `.env` and fill in:

```env
DISCORD_TOKEN=your_discord_bot_token_here
GROQ_API_KEY=your_groq_api_key_here
FASTAPI_URL=http://127.0.0.1:8000   # or your public tunnel URL
```

The bot will refuse to start if `DISCORD_TOKEN` or `GROQ_API_KEY` are missing.

## Run

The backend must be reachable on `FASTAPI_URL` first.

```bash
python bot.py
```

You should see `Logged in as <name> (id=...)` once Discord connects.

## Architecture

```
Discord user
   │  !status / !room / !usage
   ▼
bot.py (discord.py command)
   │
   ├─► api.py  ──►  FastAPI  (/devices, /usage/today/summary)
   │                  └─► returns raw JSON
   │
   └─► llm.py  ──►  Groq (llama-3.3-70b-versatile by default)
                      └─► returns friendly chat reply
   ▼
Discord reply
```

## Error handling

- **Backend down or slow** (`httpx.TimeoutException`, etc.): the bot replies with *"I'm having a little trouble reaching the facility backend right now. Give me a minute and try again."*
- **Groq errors** (rate limit, bad response, etc.): the bot replies with *"Something went wrong while formatting that update…"* — the backend is reachable, only the LLM call failed.
- **Unknown room** (`!room nope`): the bot lists the room names it does know about so the boss can self-correct.

## File layout

```
bot/
├── bot.py            # discord.py entrypoint + !status / !room / !usage commands
├── api.py            # async httpx client for the FastAPI backend
├── llm.py            # Groq humanization with strict system prompt
├── config.py         # env-var loading + validation
├── requirements.txt  # discord.py, httpx, groq, python-dotenv
├── .env.example      # template — copy to .env
├── .gitignore
└── README.md
```

## Tuning knobs (env vars)

| Var | Default | Notes |
|---|---|---|
| `GROQ_MODEL` | `llama-3.3-70b-versatile` | Any Groq chat model works. Swap for a smaller one if you hit rate limits. |
| `API_TIMEOUT` | `10` | Seconds to wait on the FastAPI call before giving up. |
| `LLM_MAX_TOKENS` | `300` | Hard cap on the humanized reply length. |
| `FASTAPI_URL` | `http://127.0.0.1:8000` | Set to your tunnel URL (e.g. `https://sad-crabs-flash.loca.lt`) when running remotely. |
